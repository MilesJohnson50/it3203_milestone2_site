HTTP Basics

HTTP is an application-layer protocol using a request/response model. Clients send a request line, headers, and an optional body; servers reply with a status line, headers, and an optional body.

Methods & Semantics

  • GET — retrieve a resource (safe, idempotent).
  • POST — submit data changes (not idempotent).
  • PUT — replace a resource (idempotent).
  • PATCH — partial update.
  • DELETE — remove a resource (idempotent).
  • HEAD — headers only (check metadata).

Headers & Caching

Common headers: Content-Type, Accept, Authorization. Cache headers: Cache-Control, ETag, Last-Modified enable conditional requests.
